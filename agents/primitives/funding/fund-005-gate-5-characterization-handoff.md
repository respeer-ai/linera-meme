Type: Primitive
Audience: Coding assistants
Authority: High

## Purpose

Close `FUND-005` with an executable characterization baseline and an exact implementation handoff for later funding iterations.

Gate 5 does not implement protocol behavior. Gate 5 binds current unsafe behavior to the funding iteration that must close it.

## Rules

- Gate 5 is a characterization and handoff artifact, not an implementation patch.
- Do not add protocol behavior changes in `FUND-005`.
- Add characterization tests only where current behavior must be locked before mutation.
- Every unsafe current behavior must map to one concrete follow-up funding iteration.
- No implementation may rely on implicit state-machine rules outside `agents/primitives/funding/fund-005-gate-4-handler-transition-table.md`.
- `FUND-006` may start only after Gate 5 is reviewed and accepted.
- Do not mark `FUND-005` complete until Gate 5 is reviewed and accepted.
- Do not include `webui/` in current frontend scope.
- Use `webui-v2/` for current frontend scope.
